Java en el codigo de Visual Studio

La compatibilidad con Java en Visual Studio Code se proporciona a través de una amplia gama de extensiones . Combinadas con el poder del núcleo VS Code, estas extensiones le brindan un editor de código liviano y de alto rendimiento que también admite muchas de las técnicas de desarrollo Java más comunes.

Este artículo le brindará una descripción general de las diferentes capacidades de Visual Studio Code para desarrolladores de Java. Para obtener un tutorial rápido sobre cómo editar, ejecutar y depurar un programa Java con Visual Studio Code, utilice el botón Tutorial de introducción a Java que aparece a continuación.

Tutorial de introducción a Java

Descripcion General

VS Code proporciona funciones de lenguaje esenciales, como finalización de código, refactorización, linting, formateo y fragmentos de código, junto con una cómoda depuración y compatibilidad con pruebas unitarias. VS Code también se integra con herramientas y marcos como Maven, Tomcat, Jetty y Spring Boot. Al aprovechar el poder de Visual Studio Code, los desarrolladores de Java obtienen una excelente herramienta tanto para la edición rápida de código como para el ciclo completo de depuración y prueba. Es una excelente opción para su trabajo en Java si busca una herramienta que:

  • Es rápido, liviano, gratuito y de código abierto.
  • Admite muchos otros lenguajes, no solo Java.
  • Le ayuda a comenzar su viaje a Java sin instalar ni aprender un IDE complejo.
  • Proporciona excelente soporte para microservicios, incluidos marcos populares, herramientas de contenedores e integración en la nube.
  • Ofrece funciones de colaboración basadas en equipos, como Visual Studio Live Share .
  • Mejora su productividad a través de IntelliSense y otras funciones de edición con reconocimiento de código.

Instalar codigo de Visual Studio para Java

Para ayudarle a realizar una configuración rápida, le recomendamos que utilice Coding Pack for Java , que es el paquete de VS Code, el Java Development Kit (JDK) y una colección de extensiones sugeridas por Microsoft. El paquete de codificación también se puede utilizar para arreglar un entorno de desarrollo existente.

Empezando

Instalar un kit de desarrollo de Java (JDK)

Un Java Development Kit (JDK) es un entorno de desarrollo de software utilizado para desarrollar aplicaciones Java. Para ejecutar Java dentro de Visual Studio Code, necesita instalar un JDK. El paquete de extensión para Java admite la versión 1.5 o superior de Java.

Para los desarrolladores nuevos en Java o en VS Code, ofrecemos algunos consejos en nuestras extensiones. Una vez que haya instalado el paquete de extensión para Java , podrá ver los consejos utilizando el comando Java: Consejos para principiantes de la paleta de comandos en VS Code.

Abra la paleta de comandos ( Ctrl+Shift+P ) y escriba "consejos de Java" para seleccionar el comando.

Trabajar con archivos fuente de Java

Puede utilizar VS Code para leer, escribir, ejecutar y depurar archivos fuente de Java sin crear un proyecto. VS Code para Java admite dos modos, ligero y estándar. El modo ligero es ideal para escenarios que solo tratan con archivos fuente. Si desea trabajar con un proyecto a gran escala, necesitará el modo estándar. Puede cambiar fácilmente del modo ligero al modo estándar, cuando sea necesario. Para obtener más información, consulte Modo ligero.

Espacios de trabajo de código VS

En Visual Studio Code, un "Espacio de trabajo" significa una colección de una o más carpetas del sistema de archivos (y sus elementos secundarios) y todas las configuraciones de VS Code que entran en vigor cuando ese "Espacio de trabajo" está abierto en VS Code. Hay dos tipos de "espacios de trabajo" en VS Code, "espacios de trabajo de carpetas" y "espacios de trabajo de múltiples raíces".

VS Code presenta un "espacio de trabajo de carpeta" cuando abre una carpeta (directorio) del sistema de archivos en VS Code.

Un "espacio de trabajo de múltiples raíces" puede hacer referencia a varias carpetas (directorios) de partes dispares del sistema de archivos y VS Code muestra el contenido de las carpetas del espacio de trabajo juntos en el Explorador de archivos .

Proyectos Java en VS Code

A diferencia de IDE como IntelliJ IDEA, NetBeans o Eclipse, el concepto de "proyecto Java" se proporciona completamente mediante extensiones y no es un concepto central en el código VS base. Cuando trabaje con "proyectos Java" en VS Code, debe tener instaladas las extensiones necesarias para trabajar con esos archivos de proyecto.

Por ejemplo, los proyectos Maven, Eclipse y Gradle Java son compatibles a través de Language Support for Java™ de Red Hat , utilizando M2Eclipse , que proporciona soporte para Maven, y Buildship , que proporciona soporte para Gradle a través de Eclipse JDT Language Server.

Espacios de trabajo de VS Code que contienen proyectos Java

Suponiendo que las extensiones de Java necesarias estén instaladas, abrir un espacio de trabajo de VS Code que contenga artefactos de Java hará que esas extensiones comprendan esos artefactos y presenten opciones para trabajar con ellos.

Edicion

Navegación de código

Java en Visual Studio Code también admite funciones de navegación del código fuente, como buscar símbolos, ver definición e ir a definición. La extensión Spring Boot Tools proporciona soporte mejorado de navegación y finalización de código para proyectos Spring Boot .

Una de las ventajas clave de VS Code es la velocidad. Cuando abra su archivo o carpeta fuente de Java, en unos segundos, con la ayuda del Modo ligero , podrá navegar por su base de código con la vista Esquema, así como con comandos como Ir a definición e Ir a referencia . Esto es especialmente útil cuando abres un proyecto por primera vez

Finalización de código

IntelliSense es un término general para las funciones del lenguaje, incluida la finalización inteligente de código (método en contexto y sugerencias de variables) en todos sus archivos y para módulos integrados y de terceros. VS Code admite la finalización de código e IntelliSense para Java a través del soporte de lenguaje para Java™ de Red Hat . También proporciona IntelliSense asistido por IA llamado IntelliCode al colocar lo que es más probable que use en la parte superior de su lista de finalización.

Mejore las terminaciones con IA

GitHub Copilot es una herramienta de finalización de código impulsada por IA que te ayuda a escribir código de forma más rápida e inteligente. Puede usar la extensión GitHub Copilot en VS Code para generar código o aprender del código que genera.

Fragmentos de codigo

Visual Studio Code admite una amplia gama de fragmentos de código Java populares para hacerlo más productivo, como clase/interfaz, syserr, sysout, if/else, try/catch , método principal estático. Utilizando información del servidor de lenguaje Java, también proporciona una vista previa del fragmento de código durante la selección.

Por ejemplo, escribir " sout " o " sysout " generará un fragmento de código para System.out.println(). De manera similar, escribir " main " o " psvm " generará un fragmento de código para public static void main(String[] args) {}

Depuracion

Debugger para Java es un depurador de Java ligero basado en Java Debug Server . Funciona con Language Support for Java™ de Red Hat para permitir a los usuarios depurar código Java dentro de Visual Studio Code.

Iniciar una sesión de depuración es fácil: haga clic en el botón Ejecutar|Depurar disponible en CodeLens de su main()función o presione F5 . El depurador generará automáticamente la configuración adecuada para usted.

Aunque es liviano, el depurador de Java admite funciones avanzadas como evaluación de expresiones, puntos de interrupción condicionales y reemplazo de código activo.

Pruebas

Con el soporte de la extensión Test Runner para Java , puede ejecutar, depurar y administrar fácilmente sus casos de prueba JUnit y TestNG.
